# 踩过的坑与解决方案

作者：张春  
整理：AI 根据项目文件整理生成  
更新时间：2026-07-07（Asia/Shanghai）
版本：v1.0

## 1. 上传 100% 不代表 OCR 完成

问题表现：用户上传 PDF 后，浏览器上传进度到 100%，但页面没有新反馈，看起来像程序没有运行。

原因：浏览器上传进度只表示文件传到后端，不表示 OCR 已完成。OCR 解析可能还需要几十秒到数分钟。

解决：前端将上传进度和 OCR 处理进度拆分，增加智能预估进度、当前阶段说明和动态过程消息。

## 2. 长耗时 OCR 阻塞 FastAPI 轮询

问题表现：复杂 PDF 处理时，前端轮询状态接口也变慢，用户无法判断任务是否还活着。

原因：重 OCR 流程如果直接在事件循环中执行，会占用服务响应能力。

解决：将主 OCR Pipeline 放到后台线程执行，让 FastAPI 继续响应状态查询。

## 3. 科研论文版面比普通 PDF 难很多

问题表现：普通 OCR 能识别文字，但科研论文中的多栏正文、表格、图片、公式、标题层级经常错位。

原因：科研论文不是简单文本流，而是复杂版面结构。仅靠通用 OCR 容易丢失阅读顺序和视觉区域关系。

解决：引入 Dolphin-v2，增加复杂科研论文路由，并在导出 HTML 时保留页面图、图片区域和表格截图。

## 4. 表格识别不能只靠 Markdown

问题表现：表格转 Markdown 后容易列错位、单元格合并丢失、边框和层级消失。

原因：Markdown 表格表达能力有限，复杂表格需要结构识别、截图或 HTML 表格共同处理。

解决：导出层保留表格截图作为兜底，同时在 Markdown 清洗和表格纠错中尽量修复简单表格。

## 5. 图片位置还原需要版面坐标

问题表现：OCR 输出有文字，但图片没有放回原文附近。

原因：很多 OCR 输出只提供文本，不提供精确图片坐标和正文锚点。

解决：使用 PDF 渲染和图片抽取脚本补充页面图、图片和表格区域。后续可进一步接入 layout block 坐标，实现更精细的锚点映射。

## 6. 大模型不是 OCR 的必要条件

问题表现：用户不确定点击 OCR 后没反应是否因为没有大模型。

原因：OCR、纠错、RAG、聊天几个概念容易混在一起。

解决：明确系统设计：OCR 主流程可以独立运行；LLM 用于纠错增强、摘要、翻译和问答。没有 LLM 时，RAG 可以采用本地检索 fallback。

## 7. 本地模型和依赖不能直接推 GitHub

问题表现：项目包含 Dolphin 权重、node_modules、上传文件、导出文件和运行日志，直接推送会体积巨大且可能泄露数据。

原因：本地开发目录混合了源码、依赖、模型、用户数据和生成结果。

解决：新增 `.gitignore`，排除 `models/`、`third_party/`、`frontend/node_modules/`、`uploads/`、`exports/`、日志和缓存，只推源码、配置、脚本、测试和文档。

## 8. Git 初次提交缺少作者身份

问题表现：`git commit` 失败，提示 `Author identity unknown`。

原因：当前仓库未配置 Git 用户名和邮箱。

解决：只在本项目设置本地 Git 身份：

```powershell
git config user.name "张春"
git config user.email "clementzhang29@users.noreply.github.com"
```

## 9. GitHub push 可能因网络失败

问题表现：第一次 `gh repo create --push` 仓库创建成功，但 push 失败，报 `Connection was reset`。

原因：本地到 GitHub 的网络连接临时不稳定。

解决：确认远程仓库已创建后，重试 `git push -u origin main`。最终推送成功。

## 10. 中文文档编码要统一

问题表现：部分旧 Markdown 在 PowerShell 中显示乱码。

原因：历史文件编码或终端编码不一致。

解决：新整理文档统一使用 UTF-8，并把关键介绍整理成 HTML，降低不同环境打开时的乱码风险。

## 11. 端口占用和多服务实例

问题表现：8080、8090、8091 可能存在不同运行实例，用户浏览器打开的页面未必是最新代码。

原因：调试过程中为了不打断已有 OCR 任务，启动了多个端口。

解决：确认当前使用端口，避免随意杀掉长任务；新版本可用 8091 验证，稳定后再切回默认端口。

## 12. 下一步建议

- 用任务队列替代内存异步任务。
- 用 WebSocket/SSE 替代轮询。
- 用数据库持久化任务和聊天记录。
- 用向量数据库提升 RAG 质量。
- 用布局坐标增强图片和表格回填。
- 给每个 OCR 引擎建立健康检查和示例验证集。
